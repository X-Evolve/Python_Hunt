Polymorphism

What is Polymorphism?

Polymorphism is a fundamental OOPs (Object-oriented programming) concept. It allows the programmer to have multiple functionalities under the same name. The term 'Polymorphism' is derived from the Greek words 'Poly' (meaning many) and 'morphism' (meaning forms).

Python provides protocols which are polymorphism under the hood. These implement consistent behavior for built-in objects of different types.

Understanding Abstract Class Based Polymorphism

Polymorphism is a core concept in object-oriented programming.

Imagine you have different animals, like a cat, a dog, and a bird. Each of them can make a sound, but they make different sounds:

  • A cat says "Meow."
  • A dog says "Woof."
  • A bird says "Chirp."

With polymorphism, you can create a common way to ask any animal to make a sound. You don't need to know which specific animal you're dealing with; you can just call the "makeSound" method on any of them.

So, when you ask an animal to "makeSound," it will respond with its own unique sound. This is what we call polymorphism - the ability of different objects to respond to the same method in their way.

In your code, you can use this polymorphic behavior to interact with objects without worrying about their specific types. This makes your code more flexible and easier to maintain.

Here its important to redefine the abstract methods in the subclass. If you don't, you'll get an error when you try to create an instance of the subclass. 

                
class Animal:
    def make_sound(self):
        # Common method for all animals
        pass

class Cat(Animal):
    def make_sound(self):
        print("Meow")

class Dog(Animal):
    def make_sound(self):
        print("Woof")

class Bird(Animal):
    def make_sound(self):
        print("Chirp")

# Create instances of Cat, Dog, and Bird
cat = Cat()
dog = Dog()
bird = Bird()

# Call the make_sound method on instances
cat.make_sound()
dog.make_sound()
bird.make_sound()

Understanding Function Based Polymorphism

What is Method Overriding?

Method overriding is a key concept in object-oriented programming. It allows a subclass to provide a specific implementation of a method that is already defined in its superclass (base class).

In function-based polymorphism, the method in the subclass has the same name, return type, and parameters as the method in the superclass, effectively replacing the superclass method with the subclass implementation.

Method overriding is used when you want to provide a specialized implementation of a method in a subclass. This allows each subclass to have its unique behavior while still adhering to a common interface defined in the superclass.

In this example, the base class "Animal" defines a method "makeSound." Subclasses "Cat" and "Dog" override this method to provide their own unique implementations.

When you create instances of "Cat" or "Dog" and call the "makeSound" method on them, each subclass responds with its specific sound. This is the essence of method overriding and function-based polymorphism. 

                    
class Animal:
def make_sound(self):
    print("Some generic sound")

class Cat(Animal):
    def make_sound(self):
        print("Meow")

class Dog(Animal):
    def make_sound(self):
        print("Woof")

# Create instances of Cat and Dog
cat = Cat()
dog = Dog()

# Call the make_sound method on instances
cat.make_sound()
dog.make_sound()

Function Overloading

Method overloading occurs when a class contains many methods with the same name. The types and amount of arguments passed by these overloaded methods vary. Python does not support method overloading or compile-time polymorphism. If there are multiple methods with the same name in a class or Python script, the method specified in the latter one will override the earlier one. Python does not use function arguments in method signatures, hence method overloading is not supported.
