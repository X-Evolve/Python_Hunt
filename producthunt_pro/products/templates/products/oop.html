OOP in Python

Object-Oriented Programming (OOP) is a fundamental programming paradigm that is widely used in Python and many other programming languages. It is a way of organizing and structuring your code to model real-world entities and their interactions. Python, as a versatile and high-level language, provides robust support for OOP concepts.

Classes

Classes are the blueprint or template for creating objects (instances). Classes define the structure and behavior of objects by specifying attributes (data) and methods (functions) that the objects created from the class will have.

Inheritance

Inheritance is a fundamental concept in Object-Oriented Programming (OOP) and is widely supported in Python. Inheritance allows you to create a new class (called a subclass or derived class) that inherits the attributes and methods of an existing class (called a base class or superclass). This mechanism promotes code reusability and enables you to create specialized classes that extend or modify the behavior of the base class. Encapsulation

Encapsulation is a concept that focuses on the bundling of data (attributes) and methods (functions) that operate on that data into a single unit, known as a class. Encapsulation promotes data hiding, controlling access to the internal state of an object, and maintaining the integrity of data. It offers several benefits, including improved code organization, security, and maintainability.

In this example, the Person class encapsulates the name and age attributes, providing getter and setter methods to access and modify them. The encapsulation ensures that attributes are not directly accessed from outside the class, promoting data integrity and control over attribute access and modification.

Polymorphism

Polymorphism is a fundamental concept in Object-Oriented Programming (OOP) that allows objects of different classes to be treated as objects of a common base class. It enables you to write code that can work with objects based on their common interface or shared behaviors, rather than their specific class or type. Polymorphism simplifies code and promotes reusability. In Python, polymorphism is achieved through method overriding and through the use of inheritance and interfaces (in a loosely-typed way).

In the honda class, you can see that the start method and stop method are given a different implementation from that of in the parent class method. Upon calling the start and stop method for the honda object we shall get the result of implementation written in honda class.
